# ระบบจัดการข้อมูลร้านค้าและสินค้ากระจาย (GitHub-Based)

ระบบนี้ถูกออกแบบมาเพื่อช่วยให้คุณสามารถจัดการข้อมูลร้านค้าและสินค้ากระจายได้อย่างมีประสิทธิภาพ โดยใช้ GitHub เป็นฐานข้อมูลหลัก และมีระบบอัตโนมัติในการลบข้อมูลสินค้าที่หมดอายุ (3 เดือน) รวมถึงรองรับการสั่งงานผ่าน AI ที่เชื่อมต่อกับ LINE Bot

## 🛠️ โครงสร้างระบบ

ระบบประกอบด้วยไฟล์หลักดังนี้:

*   `data/stores.json`: ไฟล์ JSON สำหรับเก็บข้อมูลร้านค้าและพิกัด (Latitude, Longitude) พร้อมข้อมูลเพิ่มเติมเช่น Class, Phone, Address
*   `data/inventory.json`: ไฟล์ JSON สำหรับเก็บข้อมูลสินค้ากระจาย โดยแต่ละรายการจะมีวันหมดอายุ (Expiry Date) กำกับไว้
*   `db_manager.py`: สคริปต์ Python สำหรับจัดการข้อมูลใน `stores.json` และ `inventory.json` รวมถึงการคำนวณวันหมดอายุของสินค้า
*   `.github/workflows/auto_cleanup.yml`: ไฟล์ตั้งค่า GitHub Actions เพื่อรันสคริปต์ `db_manager.py` สำหรับลบข้อมูลสินค้าที่หมดอายุโดยอัตโนมัติทุกวัน

## 🚀 วิธีการใช้งานผ่าน AI (สำหรับ LINE Bot)

เพื่อให้ AI ที่เชื่อมต่อกับ LINE Bot ของคุณสามารถสั่งงานระบบนี้ได้ คุณสามารถใช้คำสั่งในรูปแบบต่อไปนี้:

### 1. การเพิ่มหรืออัปเดตข้อมูลร้านค้า

**คำสั่ง:** `เพิ่มร้าน [รหัสร้านค้า] [ชื่อร้าน] พิกัด [ละติจูด],[ลองจิจูด] โซน [รหัสโซน] เส้นทาง [รหัสเส้นทาง] คลาส [คลาสร้าน] เบอร์โทร [เบอร์โทรศัพท์] ที่อยู่ [ที่อยู่เต็ม]`

*   **รหัสร้านค้า:** ต้องไม่ซ้ำกัน (เช่น `CUST001`)
*   **ชื่อร้าน:** ชื่อเต็มของร้านค้า
*   **พิกัด:** ละติจูดและลองจิจูด คั่นด้วยเครื่องหมายคอมม่า (เช่น `13.7563,100.5018`)
*   **รหัสโซน:** รหัสโซนของร้านค้า (เช่น `AYAPS061`)
*   **รหัสเส้นทาง:** รหัสเส้นทางของร้านค้า (เช่น `R09`)
*   **คลาสร้าน:** ประเภทของร้านค้า (เช่น `HFSS`, `HFSM`)
*   **เบอร์โทร:** เบอร์โทรศัพท์ของร้านค้า
*   **ที่อยู่:** ที่อยู่เต็มของร้านค้า

**ตัวอย่าง:**
```
เพิ่มร้าน CUST001 ร้านเจ๊หมวย พิกัด 13.7563,100.5018 โซน AYAPS061 เส้นทาง R09 คลาส HFSS เบอร์โทร 0812345678 ที่อยู่ 123 ถนนสุขุมวิท แขวงคลองเตย เขตคลองเตย กรุงเทพมหานคร 10110
```

### 2. การบันทึกสินค้ากระจาย

**คำสั่ง:** `บันทึกสินค้า [รหัสร้านค้า] [ชื่อสินค้า] จำนวน [จำนวน] เซลล์ [ชื่อเซลล์]`

*   **รหัสร้านค้า:** รหัสร้านค้าที่ต้องการบันทึกสินค้า (ต้องมีอยู่ใน `stores.json` แล้ว)
*   **ชื่อสินค้า:** ชื่อสินค้าที่กระจาย
*   **จำนวน:** จำนวนสินค้า (ตัวเลข)
*   **ชื่อเซลล์:** ชื่อผู้บันทึกข้อมูล (เซลล์)

ระบบจะคำนวณ `expiry_date` (วันหมดอายุ) โดยอัตโนมัติ +90 วันนับจากวันที่บันทึก

**ตัวอย่าง:**
```
บันทึกสินค้า CUST001 น้ำดื่ม 20 เซลล์ สมชาย
```

### 3. การเรียกดูข้อมูล (สำหรับ AI)

หากคุณต้องการให้ AI ดึงข้อมูลมาแสดงผล คุณสามารถให้ AI เรียกใช้ฟังก์ชันใน `db_manager.py` เพื่ออ่านข้อมูลจาก `stores.json` หรือ `inventory.json` และนำมาประมวลผลได้ เช่น:

*   **AI สามารถอ่าน `stores.json`** เพื่อแสดงรายชื่อร้านค้าทั้งหมด
*   **AI สามารถอ่าน `inventory.json`** เพื่อแสดงรายการสินค้ากระจายของร้านค้าใดร้านค้าหนึ่ง หรือสรุปยอดสินค้าที่กำลังจะหมดอายุ

**ตัวอย่างคำสั่งที่ AI จะใช้ (ภายในโค้ด):**
```python
# เพื่อเพิ่มร้านค้า
# db_manager.add_or_update_store("CUST001", "ร้านเจ๊หมวย", 13.7563, 100.5018, "AYAPS061", "R09", "HFSS", "0812345678", "123 ถนนสุขุมวิท")

# เพื่อบันทึกสินค้า
# db_manager.add_inventory_item("CUST001", "น้ำดื่ม", 20, "สมชาย")
```

## 🗑️ ระบบลบข้อมูลอัตโนมัติ (3-Month Expiration)

GitHub Actions ที่ตั้งค่าไว้ใน `.github/workflows/auto_cleanup.yml` จะทำงานโดยอัตโนมัติ **ทุกวันเวลาเที่ยงคืน UTC (ประมาณ 7 โมงเช้าเวลาไทย)** เพื่อตรวจสอบไฟล์ `inventory.json` หากพบรายการสินค้าใดที่มี `expiry_date` น้อยกว่าหรือเท่ากับวันปัจจุบัน ระบบจะทำการลบรายการนั้นออกจากไฟล์โดยอัตโนมัติ และทำการ `commit` และ `push` การเปลี่ยนแปลงกลับไปยัง GitHub Repository ของคุณ

## ⚠️ ข้อควรทราบ

*   **GitHub Personal Access Token (PAT):** GitHub Actions จะต้องมีสิทธิ์ในการ `write` (เขียน) ข้อมูลกลับไปยัง Repository ของคุณ เพื่อให้ระบบ Auto-cleanup ทำงานได้ หากคุณตั้งค่า PAT สำหรับ Actions ไม่ถูกต้อง อาจทำให้การลบข้อมูลอัตโนมัติไม่สำเร็จ
*   **การจัดการข้อมูล:** ระบบนี้เหมาะสำหรับข้อมูลที่ไม่ซับซ้อนมากนัก หากข้อมูลมีขนาดใหญ่ขึ้นมาก (หลายแสนรายการ) หรือต้องการความสามารถในการ Query ที่ซับซ้อน อาจต้องพิจารณาใช้ฐานข้อมูลแบบ Relational (เช่น PostgreSQL) หรือ NoSQL (เช่น MongoDB) แทน

หากมีข้อสงสัยเพิ่มเติม หรือต้องการปรับแต่งระบบส่วนใด แจ้งได้เลยครับ!
